Why DisasterMesh

When everything else fails, mesh works

Earthquakes, floods, hurricanes, and power outages take down cellular infrastructure right when people need to communicate most.

📡

BLE Mesh Network

Devices form a peer-to-peer network via Bluetooth Low Energy. Messages hop through nearby devices with no central infrastructure.

BLE GATT · Noise_XX · GATT Server
🔒

End-to-End Encrypted

HPKE Base (X25519/HKDF-SHA256/ChaCha20Poly1305) with Ed25519 signatures. Relay nodes forward ciphertext they cannot read.

RFC 9180 HPKE · Ed25519
🆘

SOS Priority Routing

PRIVATE_SOS messages receive P0 priority, 12-copy tokens, and 16-hop limits. Emergency requests get maximum delivery effort.

P0 Priority · 24h TTL
↗

Multi-hop Relay

Binary Spray-and-Wait distributes copy tokens across the network. Messages travel through up to 12–16 hops to reach their destination.

Spray-and-Wait · Token Escrow
💾

Store and Forward

Persistent SQLite storage survives process kills. Messages are held and forwarded when devices come into BLE range later.

SQLite · Android Keystore
🛡

Zero INTERNET Permission

Release builds declare no INTERNET permission. The attack surface is minimized to BLE only — no TCP, no UDP, no Wi-Fi.

BLE-only · Minimal permissions

How It Works

Message flows without the internet

01

Contact Exchange

Users exchange contact cards via QR code in person. Ed25519 public keys enable mutual verification and trusted messaging.

02

Encrypt Locally

The Rust core encrypts each message with the recipient's HPKE public key. Only the recipient's private key can decrypt it.

03

BLE Broadcast

Devices discover each other over BLE. The Noise_XX handshake secures each link. Encrypted bundles are exchanged.

04

Mesh Relay

Relay nodes carry ciphertext without reading it. Binary Spray-and-Wait distributes copies across the network.

05

Deliver + Receipt

When the bundle reaches the recipient, a signed delivery receipt flows back to the sender through the same mesh.

Message Types

Six message types, four priorities

TYPE PRIORITY TTL COPY TOKENS USE CASE
PRIVATE_SOS P0 24 hours 12 Emergency rescue request with category, description, location
DELIVERY_RECEIPT P0 7 days — Signed confirmation returned to original sender
CANCEL P0 7 days — Sender-only message revocation
CHECK_IN P1 48 hours 8 Safety status: SAFE / INJURED / NEED_ASSISTANCE / EVACUATING
LOCATION_UPDATE P1 24 hours 6 GPS or manually entered position
DIRECT_TEXT P2 72 hours 6 Personal text message, up to 2,000 Unicode scalars
