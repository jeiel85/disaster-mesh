# /goal 5 — Disaster Product and Persistent Relay

```text
/goal
Turn the multi-hop prototype into the Android disaster product defined in the requirements. Add CHECK_IN, PRIVATE_SOS, optional GPS or manual location, cancellation, trusted-recipient selection, standby/emergency/fixed-relay modes, connectedDevice foreground service, persistent notification, process and reboot recovery, battery and thermal duty-cycle policy, low-storage behavior, accessibility and explicit non-guaranteed-delivery wording. Keep relay operation possible while plaintext UI is app-locked. Do not add public chat, anonymous public SOS, files, voice, analytics, accounts or cloud services. Complete only when screen-off eight-hour tests are documented, process termination recovers committed queues, location permission is optional for SOS, battery/thermal throttling works, and the UI never labels a message delivered before a verified receipt returns.
```
